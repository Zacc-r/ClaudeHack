# UX Design Laws — Quick Reference for Design Audits

## Interaction Laws
- Fitts's Law: Tap targets minimum 44pt, important actions big and close
- Hick's Law: Max 5±2 options per screen, fewer choices = faster decisions
- Doherty Threshold: Response time under 400ms or show loading state

## Cognitive Laws
- Miller's Law: 7±2 items max in any list or group
- Jakob's Law: Follow iOS/web platform conventions
- Tesler's Law: System absorbs complexity, not user
- Goal-Gradient Effect: Show progress — people speed up near the finish

## Visual Perception (Gestalt)
- Proximity: 2-3x spacing ratio between groups
- Similarity: Same look = same meaning
- Common Region: Boundaries create visual groups
- Prägnanz: Simplify everything

## Visual Hierarchy
- Von Restorff: One standout element per screen
- Serial Position: First and last items are remembered most
- Aesthetic-Usability: Beautiful = trustworthy

## Mobile-Specific
- Thumb zone: Primary actions in bottom 1/3 of screen
- Touch targets: 44x44pt minimum
- Scroll over paginate: Let users scroll naturally
- Loading: Skeleton screens over spinners
